    01
    ENCRYPTION
    ▼

    Encryption is the process of converting readable data (plaintext) into an unreadable format (ciphertext) so only authorized people can read it

    // EXAMPLE - Caesar Cipher
    Plain: HELLO WORLD
    Shift: 3
    Encrypted KHOOR ZRUOG
    Modern encryption uses AES-256 — which would take billions of years to crack by brute force.
    HTTPS uses TLS encryption to protect everything that you send online.
    End-to-end encryption means even the app provider can't read your messages.
    🎣
    02
    PHISHING
    ▼

    Phishing is when an attacker impersonates trusted organisations to trick you into giving up passwords, credit card info, and personal information

    // SPOT THE PHISH - Is this email legit?
    From:
    Subject:
    91% of all cyberattacks begin with a phishing email
    Always hover over links before clicking - the real URL shows in the status bar
    Urgency is the #1 social engineering tactic, emails or messages may display such as "ACT NOW!" to make you fall in the trap
    🔥🧱
    03
    FIREWALL
    ▼

    A firewall is a security system that monitors and controls incoming and outgoing network traffic based on predefined rules — like a bouncer for your network

    // FIREWALL RULES SIMULATON
    ALLOW - PORT 443 (HTTPS)
    ALLOW - PORT 80 (HTTP)
    DENY - PORT 22 (SSH from external)
    DENY - PORT 3389 (RDP)
    ALLOW - PORT 25 (SMTP)
    There are two types: Network firewalls (hardware) and Host-based firewalls (software)
    Next-gen firewalls (NGFW) can inspect the actual content of packets, not just ports
    💉
    04
    SQL INJECTION
    ▼

    SQL Injection is when an attacker inserts malicious SQL code into an input field, tricking a database into revealing or deleting data

    // DEMO — Try these inputs in a fake login
    Enter credentials above
    Try: ' OR '1'='1 Try: admin'-- Try: normal user
    💉 SQL injection is still the #1 web vulnerability (OWASP Top 10)
    💉 Defence: Always use parameterised queries / prepared statements
    💉 The 2012 LinkedIn breach exposed 117M passwords via SQL injection
